# Paper Structure Templates

## Overview

This skill provides structured templates and guidelines for organizing academic research papers. Use these templates as reference when generating or reviewing paper drafts.

## Paper Draft Output Format

The standard output format for paper drafts generated by the pipeline:

```markdown
### Problem:
[Clear problem statement with context and significance]

### Rationale:
[Why this problem matters, what gap it fills, why existing solutions are insufficient]

### Necessary technical details:
[Key technical concepts, algorithms, frameworks, mathematical formulations]

### Datasets:
[Specific datasets to use, with justification and statistics]

### Paper Title:
[Compelling, specific, publication-quality title]

### Paper Abstract:
[150-300 words: Problem → Gap → Method → Results → Contribution]

### Methods:
[Detailed methodology: model architecture, training procedure, key components, algorithms]

### Experiments:
[Experimental setup: baselines, datasets, metrics, implementation details, expected results]

### Reference:
1. [Authors] "[Title]" [Venue], [Year].
2. [Authors] "[Title]" [Venue], [Year].
...
```

## Complete Research Paper Structure

For full-length research papers (8-12 pages):

### 1. Title

**Requirements:**
- Specific and descriptive (communicates the contribution)
- Avoids vague phrases like "A Study of..." or "An Investigation into..."
- Typically 8-15 words
- May include a memorable short name for the proposed method

**Effective title patterns:**
- "[Method]: [Capability] for [Task/Problem]" (e.g., "GraphFormer: Graph Transformer for Molecular Property Prediction")
- "Towards [Goal]: [Approach] via [Technique]" (e.g., "Towards Robust Object Detection via Adaptive Feature Fusion")
- "[Adjective] [Noun] for [Application]" (e.g., "Scalable Graph Neural Networks for Social Network Analysis")

### 2. Abstract

**Length:** 150-300 words

**Structure (in order):**
1. **Context/Problem** (1-2 sentences): What is the broad area and specific problem?
2. **Gap** (1 sentence): What is missing in current approaches?
3. **Method** (2-3 sentences): What does this paper propose? How does it work?
4. **Results** (1-2 sentences): What are the key findings or improvements?
5. **Contribution** (1 sentence): Why does this matter?

**Writing guidelines:**
- Be specific about the approach (name the technique, not just "a novel method")
- Include quantitative results if available (e.g., "improves accuracy by 3.2%")
- Do not use undefined acronyms
- Avoid citations in the abstract (venue-dependent)
- Write the abstract last, after the paper is complete

### 3. Introduction

**Structure (funnel approach):**

1. **Broad context** (1 paragraph): The general research area and its importance
2. **Specific problem** (1-2 paragraphs): The exact problem this paper addresses
3. **Existing approaches and limitations** (1-2 paragraphs): What has been done and why it's not enough
4. **Proposed approach** (1 paragraph): High-level description of the solution
5. **Contributions** (1 paragraph, numbered list): Clear enumeration of contributions

**Contribution list format:**
```
The main contributions of this paper are:
- We propose [method name], a [brief description] that [key advantage].
- We introduce [novel component], which [what it does and why it's novel].
- We conduct extensive experiments on [datasets], demonstrating [key result].
- We provide [additional contribution: analysis, dataset, tool, etc.].
```

### 4. Related Work

**Organization strategies:**
- **By topic**: Group related work by sub-problem or technique
- **By chronology**: Trace the evolution of approaches over time
- **By methodology**: Compare different paradigms

**For each category:**
- Summarize the approach (1-2 sentences per paper)
- Highlight strengths and limitations
- Clearly state how your work differs

**Common mistake:** Listing papers without analysis. Instead, organize around themes and show the gap your paper fills.

### 5. Methodology

**Structure:**

1. **Problem formulation** (optional): Formal mathematical definition
2. **Overview**: High-level architecture or approach diagram description
3. **Detailed components**: Each major component described in its own subsection
4. **Training/optimization**: How the model is trained
5. **Complexity analysis** (optional): Computational or space complexity

**Guidelines:**
- Use clear mathematical notation (define all symbols)
- Include architectural diagrams or flowcharts (describe them in text)
- Explain design choices and alternatives considered
- Be precise enough for reproduction

### 6. Experiments

**Structure:**

1. **Experimental setup**
   - Datasets (with statistics: size, classes, split)
   - Baselines (with citations)
   - Evaluation metrics (with definitions)
   - Implementation details (framework, hardware, hyperparameters)
   - Reproducibility information (random seeds, number of runs)

2. **Main results**
   - Comparison table with baselines
   - State-of-the-art comparison
   - Statistical significance tests

3. **Ablation study**
   - Remove/replace components one at a time
   - Show the contribution of each component

4. **Analysis**
   - Hyperparameter sensitivity
   - Qualitative examples
   - Failure cases
   - Efficiency analysis (runtime, memory, parameters)

**Baselines selection:**
- Include current state-of-the-art methods
- Include classic/foundational methods
- Include simple baselines (e.g., random, majority class)
- Aim for 5-8 baselines minimum

### 7. Results & Discussion

- Interpret the main findings
- Explain why the proposed method works
- Discuss limitations honestly
- Suggest directions for future work

### 8. Conclusion

**Structure:**
1. Summary of contributions (2-3 sentences)
2. Key findings (1-2 sentences)
3. Limitations (1 sentence)
4. Future work (1 sentence)

**Length:** 1 paragraph (typically 5-8 sentences)

### 9. References

- Numbered format (IEEE) or author-year format (APA)
- Include all cited works
- Prefer recent publications (last 3-5 years)
- Include foundational works regardless of age

## Section Length Guidelines (8-page paper)

| Section | Approximate Length |
|---------|-------------------|
| Abstract | 150-300 words |
| Introduction | 1-1.5 pages |
| Related Work | 1-1.5 pages |
| Methodology | 2-3 pages |
| Experiments | 2-3 pages |
| Conclusion | 0.5 page |
| References | 0.5-1 page |

## Academic Writing Standards

### Language and Tone
- Use formal, objective language
- Avoid first-person pronouns ("we" is acceptable in CS papers)
- Use active voice for clarity ("We propose..." not "It is proposed...")
- Be precise and specific (avoid vague quantifiers like "very", "many", "some")
- Define technical terms on first use

### Sentence Structure
- Aim for 15-25 words per sentence
- Each paragraph should have a clear topic sentence
- Use transition words between paragraphs ("Moreover", "However", "In contrast")
- Avoid nested clauses that reduce readability

### Terminology
- Use standard terminology from the field
- Define non-standard abbreviations on first use
- Be consistent with terminology throughout (don't switch between synonyms)
- Use acronyms after first full mention

### Mathematical Notation
- Define all symbols before use
- Use consistent notation throughout
- Number important equations
- Explain the intuition behind mathematical formulations
